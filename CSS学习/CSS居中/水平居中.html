水平居中         (一) 子元素 没有 指定了宽度

1.子元素无固定宽度；        不要求父元素     .child >> display: table;  margin: 0 auto;

拓展：table 类似于 inline-block 属性，但是 与 inline-block 不属于同一种东西，不会同一行

适用：子元素在 父级元素中 独占一行

优点：只需对自身元素设置，

缺点：对 IE6 7 不兼容。 且实际上 子元素还是要设置 一定的宽度 或者 不设置宽度（由内容将 宽度撑大）

2. 子元素无固定宽度；   .parent >> text-align:center;    .child >> display: inline-block; 

适用：父元素只有 这么一个子元素

优点：兼容性好

缺点：同时设置 父元素 子元素，且要求 子元素是 inline 或者 inline-block 元素

子元素位置：    要求   子元素是   行内元素 或者   行内块元素

3. 子元素无固定宽度；   .parent >>position: relative;     .child >> position: absolute ;  left: 50%;   transform: translateX(-50%)

适用：不考虑IE兼容；知道 子元素高度的情况下，父元素 设置 margin 为 子元素 拉开距离，提供位置给 子元素

优点：兼容性好

缺点：同时设置 父元素 子元素。     因为设置了 子元素 absolute 属性值,无法撑开父级元素（relative）（可以审阅元素 父元素没有被撑开）

          若一切元素都建立在 child 元素 上还行。若还要建立 其他 非child 子元素，不太行。且只兼容 IE9 以上

子元素： absloute left: 50%;

4. 使用 flex 布局，父元素： display: flex; justify-content: center;

优点：简单编写 且 易理解

缺点：兼容性不好

水平居中         (二) 子元素 指定了宽度

1. 子元素: margin: 0 auto;

2. .parent >>position: relative; .child >> position: absolute ;left: 50%;   margin-left：-（ 子元素宽度 / 2 ）

优点：兼容性好

缺点：必须知道 子元素宽度。 设置 absolute 不能撑开 父级元素 (relative)

垂直居中 （ 不要尝试用 margin: auto 0px; 因为 父子 ）

1. .parent { display:table-cell; vertical-align:middle; height:20px; }

2. .parent { display:inline-block; vertical-align:middle; height:20px; }

优点：兼容性好

缺点：子元素必须是 table-cell 或是 inline-block 属性值 ，table-cell 属性值 对 IE老版本 不友好

垂直 水平居中

1. .parent{ display:table-cell; vertical-align:middle; text-align:center; } .child { display:inline-block; }

缺点：同时设置 父元素 子元素，且要求 子元素是 inline-block 元素

2. 绝对定位 .parent{ position:relative; } .child{ position:absolute; top:50%; left:50%; transform:translate(-50%,-50%); }

缺点：累了 不说了

2. flex布局 .parent { display:flex; justify-content:center; align-items:center; }
